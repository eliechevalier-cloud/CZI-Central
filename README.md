# CZI-Central - Swimwear Brand Management

## Project Overview

CZI-Central is a comprehensive management system for our swimwear brand, handling everything from design conceptualization to production and marketing campaigns. This repository centralizes all project tracking, documentation, and workflow management for our collections.

## Directory Structure

```
/designs      - Design files and collection concepts
/marketing    - Marketing campaigns, materials, and strategies
/production   - Production tracking, supplier information, and manufacturing details
/docs         - Project documentation and guidelines
```

## Workflow

### Design Process
1. Create a new issue using the "New Design" template
2. Label with `design` and optionally `idea` for concepts
3. Design team works on concepts in the `/designs` folder
4. Review and approval process
5. Move to production phase

### Production Process
1. Create a new issue using the "Production" template
2. Label with `production` and `urgent` if time-sensitive
3. Track quantities, suppliers, and deadlines
4. Update issue with production status
5. Close when items are received/completed

### Marketing Process
1. Create a new issue using the "Marketing" template
2. Label with `marketing` and relevant additional labels
3. Plan campaigns and track budgets
4. Store materials in `/marketing` folder
5. Monitor campaign performance and update issue

## Getting Started

1. Browse existing issues to understand current projects
2. Create a new issue using the appropriate template for your task type
3. Apply relevant labels to help with organization and prioritization
4. Assign team members and set deadlines
5. Update issues regularly with progress and status changes

## Issue Templates

We have three main issue templates:

- **New Design**: For new swimwear designs and collections
- **Production**: For manufacturing and production tracking
- **Marketing**: For marketing campaigns and promotional activities

## Best Practices

- Always use the appropriate issue template when creating new tasks
- Keep issues updated with current status and progress
- Use labels consistently to maintain organization
- Link related issues together when appropriate
- Store relevant files in the appropriate directory
- Document important decisions in the `/docs` folder

## Support

For questions or assistance, reach out to the project maintainers or create an issue with the appropriate labels.
